# 传感器数据规格与 Context 化详细参考

## 概述

本文档为三类可穿戴设备（AI 眼镜、AI 耳机、AI Pin/挂坠）提供精确的传感器数据规格，包括原始数据格式、带宽计算、端侧预处理输出格式、以及从原始信号到 Context Object 字段的完整映射。

---

## 1. AI 眼镜 传感器规格

### 1.1 硬件传感器配置

| 传感器 | 典型规格 | 原始数据格式 | 采样率 | 原始带宽 |
|--------|----------|-------------|--------|----------|
| 主摄像头 | 12MP, 1920×1080 | RAW Bayer / YUV420 / H.264 | 30 fps | RAW: ~93 MB/s, H.264: ~5 MB/s |
| 深度传感器 (可选) | 640×480, 16bit | Depth Map (uint16) | 15 fps | ~9.2 MB/s |
| 麦克风阵列 | 双麦, 16bit | PCM, Little-endian | 16 kHz | 64 KB/s (双通道) |
| IMU (6-axis) | 加速度+陀螺仪 | 3×float32 + 3×float32 | 100 Hz | 2.4 KB/s |
| 磁力计 | 3-axis | 3×float16 | 50 Hz | 0.3 KB/s |
| GPS/GNSS | lat/lon/alt | NMEA sentences | 1 Hz | ~0.1 KB/s |
| Wi-Fi/BLE 扫描 | RSSI + MAC | Scan result list | 0.2 Hz | ~0.5 KB/scan |
| 环境光传感器 | lux | uint16 | 1 Hz | 2 B/s |
| 近眼显示 (输出) | microLED/光波导 | RGB framebuffer | 60 fps | N/A (输出) |

### 1.2 原始数据带宽总计

```
视觉 (H.264 编码后):    ~5,000 KB/s
深度 (若有):             ~9,200 KB/s
音频 (双通道 PCM):          64 KB/s
IMU:                        2.4 KB/s
其他:                       ~1 KB/s
────────────────────────────────────
总计 (含深度):          ~14,267 KB/s ≈ 14 MB/s
总计 (无深度):           ~5,067 KB/s ≈ 5 MB/s
```

### 1.3 端侧预处理输出

| 处理环节 | 输入 | 输出 | 压缩比 | 延迟 | 端侧模型 |
|----------|------|------|--------|------|----------|
| 视频关键帧提取 | 30fps H.264 | 1帧/2-5秒 (JPEG 50KB) | 3000:1 | <10ms | 变化检测 |
| 人脸检测 | JPEG 帧 | bbox + landmarks JSON | 50000:1 | 15-30ms | MobileFaceNet |
| 场景分类 | JPEG 帧 | 场景标签 (1 string) | ∞ | 20-50ms | MobileNetV3 |
| VAD | PCM 16kHz | speech/non-speech 标记 | ∞ | 10ms | Silero VAD |
| 活动识别 (HAR) | IMU 100Hz 窗口 | 活动标签 (1 string) | ∞ | 50ms | 轻量 CNN |
| 头部朝向 | IMU + 磁力计 | 欧拉角 (3 floats) | ~10:1 | 5ms | 传感器融合 |

### 1.4 从原始信号到 Context Object 字段的映射

| 原始信号 | 预处理 | 中间表示 | Context Object 字段 | 典型 Token 数 |
|----------|--------|----------|-------------------|-------------|
| RGB 帧 (1080p, 2MB) | 关键帧+VLM | 场景描述文本 | `scene.scene_description` | 50-150 tokens |
| RGB 帧 | 人脸检测+匹配 | 人物列表 | `scene.people[]` | 20-50 tokens/人 |
| RGB 帧 | 物体检测 | 物体列表 | `scene.objects[]` | 10-30 tokens |
| PCM 音频 (64KB/s) | VAD+ASR | 转录文本 | `audio.transcript` | 变化大 (0-500 tokens/分钟) |
| PCM 音频 | 说话人分离 | 说话人标签 | `audio.speakers[]` | 5-10 tokens |
| IMU (2.4KB/s) | HAR 分类 | 活动标签 | `motion.activity` | 3-5 tokens |
| IMU | 姿态融合 | 姿态描述 | `motion.posture` | 3-5 tokens |
| GPS (0.1KB/s) | 逆地理编码 | 地点名称 | `location.semantic_place` | 5-15 tokens |

**总计：一个典型场景的 Context Object 文本表示 ≈ 100-300 tokens**

**压缩比：原始 ~5 MB/s → Context Object ~0.5 KB/s → 约 10,000:1**

## 2. AI 耳机 传感器规格

### 2.1 硬件传感器配置

| 传感器 | 典型规格 | 原始数据格式 | 采样率 | 原始带宽 |
|--------|----------|-------------|--------|----------|
| 外部麦克风 (FF) | MEMS, 24bit | PCM, Little-endian | 16 kHz | 48 KB/s |
| 内部麦克风 (FB) | MEMS, 24bit | PCM, Little-endian | 16 kHz | 48 KB/s |
| 骨传导传感器 (可选) | 振动 | PCM 8kHz | 8 kHz | 16 KB/s |
| IMU (6-axis) | 加速度+陀螺仪 | 3×int16 + 3×int16 | 50 Hz | 0.6 KB/s |
| PPG (心率, 可选) | 光电 | uint16 | 25 Hz | 50 B/s |
| 近场 BLE | RSSI | Scan result | 0.5 Hz | ~0.2 KB/scan |
| 触摸/手势传感器 | 电容 | Event-driven | 事件触发 | ~0 (事件型) |

### 2.2 原始数据带宽总计

```
外部麦克风 (24bit PCM):     48 KB/s
内部麦克风 (24bit PCM):     48 KB/s
骨传导 (若有):              16 KB/s
IMU:                        0.6 KB/s
其他:                       ~0.3 KB/s
────────────────────────────────────
总计:                      ~113 KB/s ≈ 0.11 MB/s
```

**注意：耳机带宽远低于眼镜（~50:1），但音频处理的实时性要求更高。**

### 2.3 端侧预处理输出

| 处理环节 | 输入 | 输出 | 压缩比 | 延迟 | 端侧模型 |
|----------|------|------|--------|------|----------|
| 噪声抑制 | 双麦 PCM | 降噪后 PCM | 1:1 (质量提升) | 5-10ms | RNNoise / 自研 |
| VAD | 降噪 PCM | speech 段落时间戳 | ∞ | 10ms | Silero VAD |
| 唤醒词检测 | 降噪 PCM | boolean 触发 | ∞ | <100ms | 小型 KWS |
| 说话人分离 | 降噪 PCM | 多轨音频标签 | 1:1 | 200-500ms | ECAPA-TDNN (小型) |
| 声纹匹配 | 说话人嵌入 | person_id | ∞ | 50ms | 预计算 embedding 比对 |
| 活动识别 | IMU | 活动标签 | ∞ | 30ms | 轻量 CNN |
| 端侧 ASR (流式) | 降噪 PCM | 部分转录文本 | ~100:1 | 200-300ms | Whisper-tiny / 端侧模型 |

### 2.4 从原始信号到 Context Object 字段的映射

| 原始信号 | 预处理 | 中间表示 | Context Object 字段 | 典型 Token 数 |
|----------|--------|----------|-------------------|-------------|
| 双麦 PCM (96KB/s) | 降噪+VAD+ASR | 转录文本 | `audio.transcript` | 20-100 tokens/轮 |
| 降噪 PCM | 说话人分离 | 说话人 ID | `audio.speakers[]` | 3-5 tokens |
| 降噪 PCM | 声纹匹配 | 已知联系人 | `scene.people[]` | 10-20 tokens |
| 降噪 PCM | 情绪识别(可选) | 情绪标签 | `audio.emotion` | 3 tokens |
| IMU (0.6KB/s) | 活动识别 | 头部动作 | `motion.head_orientation` | 3-5 tokens |
| BLE 扫描 | 设备匹配 | 附近已知设备 | `scene.nearby_devices` | 5-10 tokens |

**总计：一轮对话的 Context Object ≈ 50-150 tokens**

**关键约束：从检测到"用户可能需要帮助"到输出提示，总延迟预算 <300ms**

```
延迟预算分配:
├── VAD + 降噪:          10ms
├── ASR (流式):          150ms
├── 意图检测:            30ms
├── 记忆检索:            30ms
├── LLM 生成 (1-3词):   50ms
└── TTS / 直接输出:     30ms
────────────────────────────────
总计:                   300ms
```

## 3. AI Pin / 录音挂坠 传感器规格

### 3.1 硬件传感器配置

| 传感器 | 典型规格 | 原始数据格式 | 采样率 | 原始带宽 |
|--------|----------|-------------|--------|----------|
| 麦克风 | MEMS, 16bit | PCM, Little-endian | 16 kHz | 32 KB/s |
| 加速度计 | 3-axis | 3×int16 | 25 Hz | 0.15 KB/s |
| GPS/Wi-Fi 定位 | lat/lon | 坐标 | 0.1 Hz (低频省电) | ~0.05 KB/s |
| 气压计 (可选) | hPa | float32 | 1 Hz | 4 B/s |
| 按钮/触控 | boolean | Event-driven | 事件触发 | ~0 |
| LED 指示灯 (输出) | RGB | — | — | N/A (输出) |

### 3.2 原始数据带宽总计

```
麦克风 (16bit PCM):         32 KB/s
加速度计:                   0.15 KB/s
GPS/Wi-Fi:                  0.05 KB/s
────────────────────────────────────
总计:                       ~32.2 KB/s ≈ 0.03 MB/s

全天数据量 (16小时活跃):
  音频: 32 KB/s × 16h × 3600 = ~1.85 GB (原始)
  VAD 过滤后 (~30% 有效音频): ~555 MB
  ASR 转录文本: ~50 MB (约 500万字/天)
```

### 3.3 端侧预处理输出

| 处理环节 | 输入 | 输出 | 压缩比 | 延迟 | 端侧模型 |
|----------|------|------|--------|------|----------|
| VAD | PCM 16kHz | speech 段落 | ~3:1 (过滤70%静默) | 10ms | Silero VAD |
| 场景分类 | 音频特征 | 场景标签(会议/对话/独处/通勤) | ∞ | 100ms | AudioSet 轻量分类器 |
| 说话人计数 | PCM 段落 | 人数估计 | ∞ | 200ms | 轻量 diarization |
| 关键词检测 | PCM | "记住"/"提醒"等标记 | ∞ | 50ms | KWS |
| 重要性预评估 | 音频特征+场景 | importance_score 初评 | ∞ | 100ms | 简单分类器 |

### 3.4 从原始信号到 Context Object / 记忆的映射

| 原始信号 | 预处理链 | 输出 | 存储目标 | Token 开销 |
|----------|----------|------|----------|-----------|
| PCM 16kHz (32KB/s) | VAD→场景分类→重要性评估 | 事件边界+重要性分数 | 决策:是否上传 | 0 (仅元数据) |
| 有效音频段 | 上传→云端ASR→转录 | 完整转录文本 | L4 候选 | 50-200 tokens/事件 |
| 转录文本 | LLM 结构化抽取 | 承诺/决策/待办/人物 | L4 Episodic | 30-80 tokens/条 |
| 多条 Episode | 定期归纳 | 人物画像/话题汇总 | L5 Semantic | 20-50 tokens/条 |

**全天处理链路:**

```
24h 原始音频 (~1.85 GB)
    ↓ 端侧 VAD (过滤 70% 静默)
~555 MB 有效音频
    ↓ 端侧场景分类 + 重要性预评估
~100 个事件段落 (importance > 0.4)
    ↓ 上传云端 (仅重要段落, ~50 MB)
~100 个转录文本
    ↓ LLM 结构化抽取
~30 条 Episodic Memory (importance > 0.6)
    ↓ 每周归纳
~5 条 Semantic Memory 更新
```

## 4. 跨设备对比总表

| 维度 | AI 眼镜 | AI 耳机 | AI Pin |
|------|---------|---------|--------|
| **原始带宽** | ~5-14 MB/s | ~0.11 MB/s | ~0.03 MB/s |
| **主要数据类型** | 视频+音频+IMU | 音频+IMU | 音频 |
| **端侧算力需求** | 高 (VLM关键帧) | 中 (ASR流式) | 低 (VAD+分类) |
| **Context Object token** | 100-300 /场景 | 50-150 /轮 | 30-80 /事件 |
| **压缩比 (原始→Context)** | ~10,000:1 | ~1,000:1 | ~500:1 |
| **实时性要求** | 中 (<500ms) | 极高 (<300ms) | 低 (<5s, 可异步) |
| **全天数据量** | ~250-700 GB | ~6.5 GB | ~1.85 GB |
| **全天有效 Context** | ~1000 个场景 | ~200 轮对话 | ~100 个事件 |
| **全天写入 L4 记忆** | ~50-100 条 | ~30-50 条 | ~20-30 条 |
| **端侧存储需求** | 高 (视频缓存) | 低 | 中 (音频缓存) |
| **隐私敏感度** | 极高 (视觉) | 高 (音频) | 高 (全天录音) |

## 5. Token 效率基准测试

### 5.1 VLM 场景描述 Token 开销

| 场景复杂度 | VLM 输入 (图像 token) | VLM 输出 (描述 token) | 示例 |
|-----------|----------------------|---------------------|------|
| 简单 (空房间) | 256-576 img tokens | 30-50 text tokens | "空旷的会议室，白板上无内容" |
| 中等 (2-3人对话) | 256-576 img tokens | 80-150 text tokens | "会议室，3人围坐，桌上有笔记本和咖啡" |
| 复杂 (多人+多物体) | 576-1024 img tokens | 150-300 text tokens | 详细的场景+人物+物体+空间关系描述 |

### 5.2 ASR 转录 Token 开销

| 语音类型 | 语速 (字/分钟) | Token/分钟 (中文) | Token/分钟 (英文) |
|----------|-------------|------------------|-----------------|
| 正常对话 | 150-200 字/分 | ~100-130 tokens/分 | ~150-200 tokens/分 |
| 快速讨论 | 250-300 字/分 | ~160-200 tokens/分 | ~250-300 tokens/分 |
| 演讲/报告 | 120-150 字/分 | ~80-100 tokens/分 | ~120-150 tokens/分 |

### 5.3 各层记忆的平均 Token 开销

| 记忆类型 | 单条平均 Token | 典型数量/天 | 日总 Token |
|----------|-------------|------------|-----------|
| L4 Episodic (事件记录) | 80-200 | 30-100 | 2,400 - 20,000 |
| L5 Semantic (画像/知识) | 30-80 | 5-10 (更新) | 150 - 800 |
| L6 Procedural (规则) | 20-50 | 1-2 (新增) | 20 - 100 |

## 6. 关键设计约束汇总

| 约束 | AI 眼镜 | AI 耳机 | AI Pin |
|------|---------|---------|--------|
| **端侧功耗预算** | ~500mW (持续感知) | ~100mW (持续监听) | ~50mW (持续录音) |
| **端侧 RAM** | 512MB-2GB | 128-256MB | 64-128MB |
| **端侧 Flash** | 32-128GB | 4-16GB | 8-32GB |
| **电池容量** | ~150-250mAh (轻量) | ~60-80mAh (单耳) | ~200-400mAh |
| **续航目标** | 4-6小时 (持续感知) | 6-8小时 (持续监听) | 12-16小时 (持续录音) |
| **上行带宽** | Wi-Fi(高)/BLE(低) | BLE 5.0 (~2Mbps) | Wi-Fi/BLE |
| **端侧 AI 算力** | ~2-5 TOPS (NPU) | ~0.5-1 TOPS | ~0.2-0.5 TOPS |
