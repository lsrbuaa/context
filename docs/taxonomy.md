# 跨设备用户记忆 Taxonomy

本文档定义跨设备用户记忆底座中的核心对象、记忆类型、敏感等级和使用决策规则。

## 1. 设计原则

- 不把原始数据直接等同于记忆。
- 不把向量索引直接等同于记忆库。
- 不把用户画像直接等同于可进入模型的上下文。
- 不把权限策略当作附属字段，而是作为可检索、可审计、可执行对象。
- 所有进入模型上下文的信息都必须经过任务相关性和权限过滤。

## 2. 核心工程对象

| 对象 | 定义 | 典型来源 | 是否直接进入模型 |
|---|---|---|---|
| Raw Signal | 原始设备、App、传感器或交互信号 | 心率、位置、设备日志、语音转写 | 默认不进入 |
| Event | 标准化后的单次事件 | 用户调低灯光、耳机降噪异常 | 少量按需进入 |
| Episode | 一段时间内的事件片段 | 工作日出门流程、睡前流程 | 任务相关时进入 |
| Memory | 长期有用的语义记忆 | 偏好、习惯、禁忌、任务历史 | 检索过滤后进入 |
| Profile | 稳定高层画像 | 作息类型、家庭结构、健康目标 | 谨慎进入 |
| Policy | 权限、授权、隐私和行为约束 | 不公开播报健康信息 | 相关调用必须进入 |
| Context Package | 本次模型调用上下文 | 任务、状态、记忆、约束、工具 | 是 |
| Evidence | 支持记忆的证据 | event_id、source、timestamp | 通常摘要进入 |
| Audit Record | 使用、修改、删除记录 | 检索日志、授权记录 | 不进入模型 |
| Eval Case | 评估样本 | 测试任务、期望记忆、风险检查 | 不进入生产模型 |

## 3. 记忆类型

| 类型 | 含义 | 示例 | 默认敏感度 | 使用策略 |
|---|---|---|---|---|
| preference.device | 设备使用偏好 | 用户睡前偏好低亮度暖色灯光 | low | 可用于设备控制建议 |
| preference.content | 内容偏好 | 用户不喜欢嘈杂播报风格 | low / medium | 用于回复风格和推荐 |
| habit.routine | 行为规律 | 工作日 7:40 左右离家 | medium | 需要场景授权 |
| relationship.household | 家庭成员和关系 | 家中有老人、儿童、访客 | high | 默认不进模型，需明确必要性 |
| state.device | 设备状态记忆 | 耳机近期降噪异常 | low / medium | 可用于售后诊断 |
| state.environment | 环境状态 | 卧室夜间常见温湿度 | low / medium | 任务相关时使用 |
| task.history | 历史任务和未完成事项 | 上次排查到路由器问题 | medium | 用于连续任务 |
| constraint.user | 用户明确禁忌 | 不要再推荐咖啡 | low / medium | 优先级高于推断偏好 |
| policy.privacy | 隐私和授权策略 | 不在公共场景播报健康数据 | high | 相关调用必须进入 |
| correction.feedback | 用户纠错记录 | “不是我，是家人调的灯” | medium | 立即影响旧记忆 |

## 4. 敏感等级

| 等级 | 数据范围 | 默认动作 | 进入模型条件 |
|---|---|---|---|
| low | 设备偏好、亮度、音量、常用模式 | 可沉淀为记忆 | 任务相关、无冲突 |
| medium | 位置、作息、家庭场景、联系人关系 | 最小化、场景授权 | 当前任务必要且用户授权 |
| high | 健康、身份、儿童、财务、私密对话 | 默认不进模型 | 显式授权、必要性证明、严格审计 |
| forbidden | 用户拒绝或法律禁止的数据 | 不采集、不推断、不使用 | 永不进入 |

## 5. 生命周期状态

| 状态 | 含义 | 系统行为 |
|---|---|---|
| candidate | 候选记忆 | 等待评分或更多证据 |
| active | 当前可用 | 可被检索，但仍需权限过滤 |
| pending_confirmation | 待确认 | 存在冲突或证据不足，不直接使用 |
| active_with_override | 有近期覆盖 | 旧偏好仍保留，但当前新信号优先 |
| deprecated | 过期或长期未验证 | 默认不主动召回 |
| rejected | 被判定错误 | 避免再次生成同类记忆 |
| deleted | 用户删除 | 必须不可恢复、不可使用 |

## 6. 使用优先级

当多条记忆或策略冲突时，推荐优先级为：

1. 法律和安全约束。
2. 用户显式拒绝或删除。
3. 当前场景权限策略。
4. 用户近期明确表达。
5. 用户长期显式偏好。
6. 多次重复的行为规律。
7. 单次弱推断或低置信记忆。

这意味着系统不应简单平均新旧记忆。近期明确表达应能临时覆盖长期习惯；用户删除应高于任何推断。

## 7. Context Package 最小结构

Context Package 是本次模型调用的最终输入，不是记忆库的完整镜像。

```text
Context Package
- current_task: 当前任务目标
- current_state: 当前设备、环境、时间和工具状态
- relevant_memories: 少量相关记忆
- evidence_summary: 必要证据和置信度
- policy_constraints: 隐私、授权和行为约束
- output_contract: 模型输出格式和动作边界
```

### 7.1 进入规则

| 信息 | 是否进入 | 说明 |
|---|---|---|
| 当前任务 | 必须 | 没有任务就无法判断相关性 |
| 当前状态 | 按需 | 只保留会影响决策的状态 |
| 用户记忆 | 少量进入 | 只放入能改变模型决策的记忆 |
| 原始日志 | 默认不进入 | 只用于追溯和证据 |
| 高敏数据 | 默认不进入 | 除非满足显式授权和必要性 |
| 权限策略 | 相关调用必须进入 | 用于约束模型行为 |

## 8. 反模式

- 把所有历史对话拼进上下文。
- 把向量库召回结果不经过权限过滤直接交给模型。
- 把用户画像作为稳定事实长期不更新。
- 用单次行为推断长期偏好。
- 删除自然语言记录但保留向量、摘要或缓存。
- 只评估回答质量，不评估隐私泄露和过期记忆误用。
