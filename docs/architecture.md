# 系统架构

本文档描述 User Memory Context Engineering 的系统边界、核心模块和数据分层。

## 目标

系统目标是在每次 Agent 调用模型前，生成一个高质量上下文包：

```text
当前任务目标
+ 当前设备和环境状态
+ 与任务相关的少量用户记忆
+ 必要证据和置信度
+ 隐私和安全约束
+ 可调用工具状态
= 本次模型调用上下文
```

这个上下文包应满足：

- 任务相关，而不是历史完整。
- 信息短而结构清晰。
- 带有时间、证据和置信度。
- 能处理新旧偏好冲突。
- 严格遵守当前场景下的权限和隐私约束。

## 用户记忆分层

| 层级 | 内容 | 示例 | 默认使用方式 |
|---|---|---|---|
| L0 原始数据层 | 传感器、日志、语音、位置、设备状态 | 心率、位置轨迹、设备日志 | 默认不进模型 |
| L1 结构化事件层 | 规范化后的单次事件 | 用户 7:42 离家；调低卧室灯光 | 少量按需进入 |
| L2 情景片段层 | 一段时间内的事件组合 | 工作日早晨出门流程 | 按任务进入 |
| L3 语义记忆层 | 长期偏好和习惯 | 睡前偏好低亮度暖色灯光 | 检索后进入 |
| L4 用户画像层 | 稳定高层画像 | 作息类型、家庭结构、健康目标 | 谨慎进入 |
| L5 策略与权限层 | 隐私、安全、授权、禁忌 | 不在公共场景播报健康数据 | 相关调用必须进入 |

核心原则：

> 原始数据用于追溯，结构化事件用于分析，语义记忆用于个性化，策略权限用于约束 Agent。

## 核心模块

| 模块 | 职责 | 关键问题 |
|---|---|---|
| 数据接入 | 汇聚设备、App、云端和用户交互数据 | 数据是否可靠、是否授权 |
| 身份归因 | 判断事件属于哪个用户、家庭成员、访客或设备主体 | 共享设备、多人家庭和误触如何处理 |
| 清洗脱敏 | 去噪、去重、脱敏、规范化 | 哪些数据不应进入记忆系统 |
| 事件结构化 | 把原始数据转成标准事件 | 如何统一多设备数据语义 |
| 记忆抽取 | 从事件中抽取长期有用信息 | 什么值得记住 |
| 记忆评分 | 判断记忆价值和可信度 | 如何处理频率、新近性、重要性 |
| Evidence Ledger | 记录记忆来源、授权快照和派生索引 | 如何解释、审计、删除和失效 |
| Policy Engine | 判断当前场景是否允许使用某条记忆 | 权限、敏感度、场景和动作风险如何组合 |
| 记忆库 | 存储结构化记忆和索引 | 如何支持全文、向量、图谱、审计和删除 |
| 检索重排 | 为当前任务找到最相关记忆 | 如何避免取错、漏取、取太多 |
| Context Composer | 组装最终上下文 | 如何控制 token、顺序和格式 |
| 反馈更新 | 根据用户反馈和任务结果更新记忆 | 如何纠错、遗忘、处理冲突 |

## 检索与组装流程

```text
用户请求 / 当前任务
 -> 任务分类
 -> 场景权限判断
 -> 身份归因
 -> 初步召回记忆
 -> 时间、权限、敏感度过滤
 -> rerank
 -> 冲突检测
 -> context 预算裁剪
 -> 输出给 Context Composer
```

推荐排序因子：

1. 与当前任务直接相关。
2. 用户显式确认过。
3. 最近仍被验证。
4. 置信度高。
5. 敏感度低。
6. 信息短且清晰。
7. 无冲突或冲突已解决。

## Context Composer 契约

Context Composer 的输入应包含：

- 当前任务和用户请求。
- 当前设备、环境和工具状态。
- 候选记忆列表。
- 每条记忆的证据、置信度、敏感等级和使用权限。
- 当前 token 预算。
- 输出格式约束。

Context Composer 的输出不是自由文本，而是一个可审计的 Context Package。推荐最小契约：

```json
{
  "context_package_id": "ctx_001",
  "task": {},
  "current_state": {},
  "memories": [],
  "policies": [],
  "evidence_summaries": [],
  "redaction_report": {},
  "token_budget": {},
  "audit_refs": []
}
```

给模型看的最终文本可以由 Context Package 渲染，例如：

```markdown
## 当前任务
用户希望系统在睡前自动调整卧室环境。

## 当前状态
- 时间：22:35
- 用户位置：家中卧室
- 当前卧室灯光：亮度 70%，冷色温
- 当前空调：26°C

## 相关用户记忆
- 用户睡前偏好卧室灯光低亮度、暖色温。置信度：0.84，最近验证：2026-06-15。

## 隐私与行为约束
- 不要主动播报健康数据。
- 如果需要改变多个设备状态，应先给出简短确认。
```

## 隐私默认值

- 高敏数据默认不进入模型上下文。
- 场景授权不足时，记忆只能用于本地规则判断，不应传给模型。
- 每条被使用的记忆都应能追溯到证据。
- 用户删除的记忆不得通过摘要、向量或缓存再次恢复。
- 家庭多人场景必须区分身份、权限和可见范围。
