# 案例研究：AI 耳机 / 耳戴设备

## 产品定位

对话增强型助手，通过实时语音理解和低打扰提示辅助用户社交和工作对话。

## Context 管理链路

```
┌─────────────┐     ┌──────────────┐     ┌─────────────────┐
│  麦克风输入  │────→│  语音识别     │────→│  对话理解        │
│  (用户+环境) │     │  + 说话人分离 │     │  + 意图检测      │
└─────────────┘     └──────────────┘     └────────┬────────┘
                                                   │
                    ┌──────────────┐               │
                    │  对话记忆     │←──────────────┤
                    │  人物画像     │               │
                    │  话题历史     │               ↓
                    └──────┬───────┘     ┌─────────────────┐
                           │             │  介入决策        │
                           └────────────→│  是否需要帮助？  │
                                         │  打扰代价多大？  │
                                         └────────┬────────┘
                                                   │
                                                   ↓
                                         ┌─────────────────┐
                                         │  输出执行        │
                                         │  1-3 词低声提示  │
                                         └─────────────────┘
```

## 典型场景

### 场景 A：会议中的信息辅助

**触发条件**: 对方提到一个项目名称，用户停顿（可能在回忆）

**Context Assembly**:
- Audio: 对方说 "Alpha 项目的进度怎么样了"
- Episodic Memory: Alpha 项目上周五完成了第一阶段
- User State: 短暂停顿，可能在检索信息

**输出**: 低声提示 "上周五完成一阶段"

### 场景 B：社交场合人名提醒

**触发条件**: 有人走近打招呼，用户可能不记得名字

**Context Assembly**:
- Audio: 声纹匹配到已知联系人
- Semantic Memory: 王总，合作方，上次见面是上月展会

**输出**: 轻声提示 "王总，展会"

### 场景 C：语言辅助

**触发条件**: 对方使用外语，用户可能需要翻译

**Context Assembly**:
- Audio: 检测到非母语内容
- User Preferences: 用户设置了英语辅助模式
- Conversation Context: 当前讨论技术方案

**输出**: 关键术语的快速翻译提示

## 关键设计约束

### 极短输出原则

| 约束 | 原因 |
|------|------|
| 最多 1-3 个词 | 不打断用户当前对话流 |
| 延迟 < 300ms | 信息必须在语境窗口内到达 |
| 可忽略性 | 用户可以不理会，不影响对话 |
| 无需确认 | 不要求用户做出反应 |

### 介入决策模型

```
ShouldIntervene =
    UserNeedProbability      (用户需要帮助的概率)
  × InformationValue         (提供信息的价值)
  × (1 - InterruptionCost)   (打扰的代价)
  × (1 - SocialRisk)         (社交尴尬风险)
```

**不应介入的场景**:
- 用户正在倾听，无需回应
- 对话情绪紧张，任何提示都可能分散注意力
- 用户已经在流畅回答

## 隐私治理

- 对话内容默认不存储原始音频
- 仅保留结构化摘要（话题、人物、要点）
- 他人语音需特别处理：不存储声纹、不存储原文
- 用户可设置"隐私模式"：完全停止记录
