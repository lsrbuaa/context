# Generative Research GUI Blueprint

本目录保存“智能硬件时代的 Context Engineering”研究内容转化为 Generative User Interface 的过程文件。

## 当前目标

先完成 GUI 对应的内容设计，再考虑如何通过前端、动效、imagegen 底图和交互逻辑实现。

这个 GUI 的出发点不是把研究内容完整、拥挤地装进去，而是帮助其他人简单、清晰、形象地理解我们的研究内容、成果和关键结论，并能直接指导实践。

本阶段不直接进入高保真视觉或代码实现，而是明确：

1. 总研究世界如何分区。
2. 每个区域承载哪些研究内容。
3. 用户点击一个节点后进入什么“深挖界面”。
4. 每个深挖界面展示哪些机制、证据、案例和开放问题。
5. 哪些界面适合用 imagegen 生成高质量底图。
6. 哪些文字、连线、控件必须由前端叠加，不能烘焙进图片。

## 核心原则

- GUI 不是文档目录，而是一个可探索的研究世界。
- 点击节点不是展开文字，而是进入一个新的语义空间。
- 每个界面只服务一个核心理解目标，避免把文档细节完整搬进去。
- 默认展示“关键结论 + 为什么重要 + 实践启发”，细节通过证据层和原文入口逐级下钻。
- 每个深挖界面都要给出一个明确的实践指导：产品/架构/研究下一步应该怎么做。
- 视觉底图负责氛围、空间和隐喻；真实文字、节点、连线、控件由前端叠加。
- 所有界面都必须能回溯到研究材料、证据强度和开放问题。
- 第一版先保证内容逻辑和交互结构正确，再追求视觉复杂度。
